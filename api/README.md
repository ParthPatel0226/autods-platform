# AutoDS API

FastAPI service exposing agents/, core/, evaluation/, etc. as HTTP endpoints
for the Next.js frontend.

## Hard rules

1. NEVER modify backend modules. Compose them via api/services/.
2. State I/O goes through api/services/state_service.py — which wraps
   session/session_manager.py. No other service touches state directly.
3. Backend exceptions raised from core/exceptions.py are mapped to HTTP
   codes by the central exception handler in main.py.

## Adding an endpoint

1. Add a Pydantic schema in api/schemas/
2. Add a service function in api/services/
3. Add a route in api/routes/ that calls the service
4. Register the router in api/main.py
5. Add a smoke test in api/tests/

## Running

```bash
cd api
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

## Deployment

Render. Build with the Dockerfile in this folder. Set env vars:
- SUPABASE_URL, SUPABASE_KEY
- GEMINI_API_KEY
- JWT_SECRET (generate a long random string)
- ALLOWED_ORIGINS (comma-separated)
- ALLOWED_ORIGIN_REGEX (for Vercel preview deployments)

## Endpoints

See http://localhost:8000/docs (auto-generated OpenAPI).

The frozen OpenAPI contract is at api/openapi.json (regenerated by CI).
